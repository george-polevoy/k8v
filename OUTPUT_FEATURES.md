# Output Features

## Overview

k8v now includes comprehensive output visualization for each node. When you run a computation, you can see both text output (stdout/stderr) and graphics output for each node.

## Features

### Text Output
- Captures all `print()` and `log()` statements from node execution
- Displays errors and exceptions
- Shows in a collapsible panel with monospace font
- Can optionally render non-empty node text output directly on canvas as an attached HTML `<pre>` overlay
- Per-node canvas controls let you toggle text overlays, cap visible lines, or keep a scrollable multiline view
- Dark theme for better readability

### Graphics Output
- Displays images generated by nodes
- Graphics metadata is returned inline (`id`, `mimeType`, mip-level dimensions)
- Binary image payload is fetched separately by graphics id
- PNG outputs are persisted with mip-map downsample levels for size-aware retrieval
- Python runtime can emit PNG from raw bytes, raw PNG base64, `outputPng(...)`, or figure/image objects
- Python node images are projected directly on the canvas below the node card (no in-card frame/padding)
- Automatically scales to fit the panel
- Collapsible panel for space management

## Usage

1. **Select a node** on the canvas
2. **Run the graph** using the ▶ button in the toolbar
3. **View outputs** in the Output Panel on the right side:
   - Text Output: Shows all printed text and errors
   - Graphics Output: Uses the graphics id + metadata and requests an appropriate mip level by pixel budget
4. In the node panel, enable **Display Text Outputs** if you want non-empty text output rendered on canvas for that node.
   - This persists directly on the node as flat config keys: `displayTextOutputs`, `textOutputMaxLines`, and `textOutputOverflowMode`.
5. For nodes with canvas output enabled, text is rendered below the node card as a synchronized HTML `<pre>` overlay.
6. For `python_process` nodes with graphics output, the latest image is projected on canvas directly below that node card (or below the text overlay if one is visible).

## Code Examples

### Text Output Example
```javascript
// In a node's inline code:
print("Hello, world!");
print("The result is:", inputs.value);
outputs.result = inputs.value * 2;
```

### Graphics Output Example
```javascript
// Generate a simple graphics output (base64 encoded)
const canvas = document.createElement('canvas');
canvas.width = 200;
canvas.height = 200;
const ctx = canvas.getContext('2d');
ctx.fillStyle = 'blue';
ctx.fillRect(0, 0, 200, 200);
const dataUrl = canvas.toDataURL();
outputGraphics(dataUrl); // or outputImage(dataUrl)
```

### Python PNG Output Example
```python
# Option 1: raw PNG bytes (auto-converted to data URL)
import base64
png_bytes = base64.b64decode("iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAQAAAC1HAwCAAAAC0lEQVR42mP8/x8AAwMCAO7Z8S8AAAAASUVORK5CYII=")
outputGraphics(png_bytes)

# Option 2: raw PNG base64 string
outputPng("iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAQAAAC1HAwCAAAAC0lEQVR42mP8/x8AAwMCAO7Z8S8AAAAASUVORK5CYII=")
```

## UI Layout

The Output Panel is positioned on the right side of the screen, next to the Node Panel:
- **Left**: Canvas with nodes
- **Middle**: Node Panel (for editing node properties)
- **Right**: Output Panel (for viewing results)

Both text and graphics sections are:
- **Collapsible**: Click the header to expand/collapse
- **Stacked**: Text output on top, graphics below
- **Auto-refreshing**: Updates when you run computations
